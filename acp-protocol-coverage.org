* High-Level Completion
- Core baseline agent-side flow (initialize/auth/new/prompt/update/end-turn): largely implemented.
- Biggest missing protocol surface is transcript replay completeness for multi-session (=session/load=), plus client capabilities (file system, terminal).
- Practical next milestone: finish =session/load= history rendering (replay update taxonomy), then move to fs/terminal and fuller tool-call rendering.

* Suggested Implementation Order
1. Finish =session/load= transcript replay: implement rendering for replay update types (at minimum =user_message_chunk=, plus any assistant chunk variants the agent emits during replay).
2. Improve session picker UX: paging for =session/list= (=nextCursor=), and display more metadata (cwd, updatedAt, tags).
3. Optional: add =session/close= if agent supports it (and a cleanup command).
4. File-system methods (read/write) with strict absolute path checks.
5. Terminal capability methods.
6. Richer =session/update= support for tool calls and plans (full lifecycle, not only =rawOutput=).

* UI Notes (current state)
- Chat buffer major mode: =cursor-acp-chat-mode= derived from =markdown-mode= (when installed).
- Keybindings: ACP keys are provided via an emulation keymap so they continue to work even under markdown.
- Permission UX:
  - Requests render in chat with command fenced blocks and a bold Reason section.
  - Minibuffer prompt waits indefinitely.
  - If the prompt is cancelled, the request stays pending and can be retried via =C-c C-a= (=cursor-acp-reprompt-permission=).
- Tool call UX:
  - When =tool_call_update.rawOutput= is present, it is printed in fenced =text= blocks, split per key (e.g. stdout/stderr).
